# Security Policy

## Supported version

Corrections are applied to the current default branch. This repository contains an offline analysis and no deployed service.

## Report a vulnerability

Please use the repository's **Security** tab to submit a private security advisory. Include the affected file, a minimal reproduction, and the potential impact. Do not disclose a suspected vulnerability in a public issue before it has been reviewed.

## Data safety

The committed snapshot contains public inspection attributes needed for the analysis. It excludes establishment names, street addresses, coordinates, license numbers, violation narratives, credentials, and local file paths.

Do not add API tokens, personal data, private exports, or local environment details to issues, notebooks, or pull requests. A Socrata application token is not required by the bounded retrieval script and should not be committed.
